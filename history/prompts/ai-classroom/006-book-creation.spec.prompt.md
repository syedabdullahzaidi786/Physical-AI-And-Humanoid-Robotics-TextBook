## Technical Decisions & Trade-offs

### 1. Autogenerated vs. Manual Sidebar
**Decision:** Use autogenerated sidebar from `sidebars.ts`  
**Rationale:** Reduces maintenance burden; filesystem structure drives navigation  
**Trade-off:** Less control over ordering; mitigated by naming convention (00-, 01-, etc.)

### 2. Broken Links Handling
**Decision:** Set `onBrokenLinks: 'ignore'` in config  
**Rationale:** External cross-references (simulation artifacts, GitHub repos) are not part of Docusaurus build  
**Trade-off:** May miss some internal link errors; mitigated by manual verification

### 3. Document Scope
**Decision:** Target 3,500–5,000 words total across all sections  
**Rationale:** Accessible for educational use without overwhelming readers  
**Trade-off:** Full long-form versions relegated to appendices/research folder; mitigated by links to `research/ai-classroom/`

### 4. Case Study Condensation
**Decision:** Produce 500-word summaries in book; full versions stored in `research/ai-classroom/cases/`  
**Rationale:** Book focuses on synthesis; researchers access details from research folder  
**Trade-off:** Book readers need GitHub access for full case studies
